# QA Prompt Library

## Purpose
This repository contains reusable prompts for QA engineers to improve testing workflows using AI tools. It provides a structured approach for manual testing, exploratory testing, and bug analysis.

## Repository Structure

- `manual-testing/` – Contains prompts for manual testing practices:
  - `test-design.md` – Functional test case generation
  - `exploratory-testing.md` – Guided exploratory testing scenarios
  - `bug-analysis.md` – Deep bug analysis and triage

## How to Use

1. Copy the prompt text from the relevant `.md` file.
2. Paste it into your AI tool.
3. Replace placeholders such as `[PLACE FEATURE DESCRIPTION HERE]` or `[PLACE BUG DESCRIPTION HERE]`.
4. Generate results in a structured format for test planning, exploratory testing, or bug triage.

## How to Extend

- Add new categories, such as `automation-testing` or `performance-testing`.
- Add additional prompt files following the same Markdown structure.
- Maintain consistent headings and output tables for clarity.

## Professional Notes

- Keep prompts concise but context-rich.
- Link prompts to test cases, requirements, or known defects where possible.
- Use this library as a foundation for AI-assisted QA workflows, improving consistency and efficiency.
